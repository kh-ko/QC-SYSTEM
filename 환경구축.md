## 2. 서버 내부 환경 구축 (SSH 접속)

### 2-1. 가상 RAM(Swap) 설정 (메모리 부족 방지)
```bash
# 1. 4GB 빈 공간 스왑 파일 생성
sudo fallocate -l 4G /swapfile
# 2. 권한 설정
sudo chmod 600 /swapfile
# 3. 스왑 포맷 및 활성화
sudo mkswap /swapfile
sudo swapon /swapfile
# 4. 재부팅 시에도 영구 유지되도록 등록
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 2-2. Docker 설치

```bash
# 1. 시스템 업데이트
sudo apt-get update && sudo apt-get upgrade -y
# 2. 도커 자동 설치 스크립트 실행
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
# 3. 도커 관리자 권한 부여 (재접속해야 완벽히 적용됨)
sudo usermod -aG docker $USER  
# 4. 설치 확인
docker --version
```

## 5-2. 사용자 테이블 생성 및 자동 연동(Trigger) 쿼리 실행
### 1. [Default Project] > 좌측 메뉴 [SQL Editor] > [New Query] 클릭
### 2. 아래 SQL을 복사하여 넣고 우측 하단 [RUN] 클릭 (Google 로그인 시 자동으로 public.users에 회원 정보 저장)
```sql
-- 1. 사용자 정보를 담을 public 테이블 생성
create table public.users (
  id uuid references auth.users not null primary key,
  email text,
  name text,
  avatar_url text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. 외부 읽기 허용(선택)
alter table public.users enable row level security;
create policy "Public profiles are viewable by everyone." on public.users for select using ( true );

-- 3. 구글 인증 정보 발생 시 데이터를 복사하는 함수
create function public.handle_new_user()
returns trigger language plpgsql security definer set search_path = public
as $$
begin
  insert into public.users (id, email, name, avatar_url)
  values (
    new.id,
    new.email,
    new.raw_user_meta_data->>'full_name',
    new.raw_user_meta_data->>'avatar_url'
  );
  return new;
end;
$$;

-- 4. 자동으로 위 함수를 실행하는 트리거 달기
create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();
```
